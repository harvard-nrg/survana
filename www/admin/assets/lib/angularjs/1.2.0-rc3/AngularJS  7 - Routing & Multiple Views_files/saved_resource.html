                    @Fredrik, a module's config() method (if any) executes when a module is loaded (during bootstrap).  This method can inject "providers" such as $routeProvider, $locationProvider, etc.  Providers can not be injected after bootstrap (i.e., when the injector is used to create services or retrieve an existing service instance).  E.g., when a controller, a factory, or a service uses dependency injection, it can not inject a provider like $routeProvider.  This topic is discussed more on the Modules page, section "Module Loading & Dependencies".

                                                              For those reading the discussion and wondering the same thing, using HTML5 push state effects the URL of the browser. This works fine if manipulated only by JS, but if you refresh the page or bookmark, /phones will be requested by the browser. If using a simple file system web server (like in the tutorial), those paths won't resolve to anything useful and will most likely return 404 errors.

                                                              The server would have to be trained to treat these routes and serve the index.html page instead. Avoiding hashbangs can be tricky - something like connect.js 'static' route matcher can match static resources, something like '/api' to match data calls and serve index.html for the rest. Use caution though - you might be surprised when some unmatched route gives you the app's index.html page.

                                                                                                                                              Create a service and inject it into both controllers. The first controller would store the data in the service, via some service method. The second controller could then retrieve the data. (Services are singletons, so the second controller will not create a new service – it will use the existing service.)

                                                                                                                                                                          Actually if you examine the DOM then you would see that the PhoneListCtrl always shows up as an object on the global window. One way I found of reducing the scope of it was to do something like this

                                                                                                                                                                          In app.js do change this line

                                                                                                                                                                          when('/phones', {templateUrl: 'partials/phone-list.html', controller: PhoneListCtrl}).

                                                                                                                                                                          to (simply adding '' around PhoneListCtrl, nothing else)

                                                                                                                                                                          when('/phones', {templateUrl: 'partials/phone-list.html', controller: 'PhoneListCtrl'}).

                                                                                                                                                                          In controller.js do this

                                                                                                                                                                          var app = angular.module('phonecat');
                                                                                                                                                                          app.controller('PhoneListCtrl',['$scope','$http',function($scope,$http) {
                                                                                                                                                                          //rest of the controller code stays the same here

                                                                                                                                                                          }]);

                                                                                                                                                                          If you declare the controller like this then it doesn't show up on the DOM. However your DOM does now has an app variable but I guess this variable always exist already I am just making it explicit. I am by no means an expert, just sharing what I observed.
